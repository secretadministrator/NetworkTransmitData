# DirectTransfer

DirectTransfer 是一个 Windows 双机点对点文件传输工具。它以 C++17 和 Win32 API 实现图形界面，使用自定义的 TCP 二进制协议传送文件；不依赖 SMB、Windows 文件共享、Robocopy 或 Windows 登录凭据。

当前版本：1.1.0；协议版本：v5。

## 适用场景

- 两台 Windows 电脑处于同一局域网，或以网线直连且 IP 可互通。
- 需要在不创建共享文件夹的前提下传送一个目录。
- 接收端能够选择安全复制、覆盖复制或同步镜像策略；当前 v5 主链路只有同步镜像会改变传输后的目录清理行为。

> 传输使用会话令牌防止误连，但没有提供传输加密或强身份认证。请只在受信任的网络中使用。

## 使用方法

1. 在两台电脑上启动 `DirectTransfer.exe`。如使用直连网线，请先确保两端网络地址可互通；程序中的静态 IP 设置仅是可选的管理员辅助功能。
2. 在接收端点击“接收文件”，选择目标目录，按需选择传输模式，然后点击“开始接收”。接收端会生成会话令牌并开始等待发现及 TCP 连接。
3. 在发送端点击“发送文件”，选择源目录。等待自动发现接收端；发现失败时，输入接收端 IP 并点击“使用 IP”。
4. 在发送端点击“开始发送”。界面下方会显示扫描、传输、速度、预计时间和日志。
5. 传输完成后检查接收目录和日志。若发送端提示连接中断，可点击“重新连接”继续本次会话中尚未提交的文件。

### 传输模式

| 模式 | 行为 |
| --- | --- |
| 安全复制 | 当前 v5 主链路会写入全部源文件；名称对应旧计划模块的跳过语义，尚未接入 v5 数据通道。 |
| 覆盖复制 | 当前 v5 主链路同样会写入全部源文件并替换同名文件。 |
| 同步镜像 | 完成源文件传输并提交会话后，删除目标目录中源目录没有的文件。此模式具有破坏性。 |

## 网络与防火墙

- UDP `49321`：接收端响应发送端的发现请求；发送端可广播，也可按手动输入的 IP 定向发现。
- TCP `49321`：实际的握手、目录和文件传输。
- 两端的 Windows 防火墙或第三方防护软件必须允许以上端口的入站/出站通信。程序不会自行添加防火墙规则。
- 程序不修改注册表，也不要求管理员权限；只有设置静态 IP 或恢复 DHCP 时需要以管理员身份运行。

## 传输机制

- 握手验证应用标识、协议版本和接收端生成的会话令牌。
- 文件目录使用二进制格式传送并带 CRC32；接收端拒绝绝对路径和 `..` 等不安全相对路径。
- 64 KiB 及以下的小文件按批传送：每批最多 512 个文件或 8 MiB，最多同时等待 4 个批次确认。
- 较大文件按 1 MiB 分块流式传送；文件完成时校验 CRC32。
- 接收端先写入 `.dtpart`，校验成功后才替换正式文件。小文件写入会按目标磁盘特性使用 1、2 或 4 个工作线程。
- 发送端在连接中断后以 1 秒至 15 秒的退避间隔重连；同一接收端进程已经提交的文件会按文件 ID 跳过。未提交的大文件会重新传输，不是字节级断点续传。
- 发送前及发送后会比较源文件的大小和最后修改时间，避免传输过程中源文件发生变化而被静默接受。

## 构建

要求：Windows、CMake、Ninja、支持 C++17 的 MinGW-w64 g++。项目默认使用静态链接。

```powershell
cmake -G Ninja -DCMAKE_CXX_COMPILER=g++ -B build
cmake --build build
.\build\DirectTransfer.exe
```

`external/nlohmann/json.hpp` 已随仓库提供。若该文件缺失，CMake 会尝试下载 nlohmann/json 3.11.3；离线构建时请手动放回该文件。

## 项目结构

```text
src/
├── main.cpp / MainWindow.*       程序入口、窗口和跨线程 UI 通知
├── RoleSelectPage.*              角色选择页
├── SenderPage.* / ReceiverPage.* 发送端与接收端配置页
├── NetworkDiscovery.*            UDP 发现与定向发现
├── TransferSession.*             TCP 会话、协议状态机、重连和文件 I/O
├── TransferProtocol.*            通用二进制包格式
├── FileScanner.* / ExcludeRules.* 目录扫描与默认排除规则
├── ProgressTracker.*              速度、进度和 ETA
├── ConsoleDashboard.* / ConsoleLogView.* 传输仪表板与日志视图
├── IpConfigurator.*              可选的静态 IP / DHCP 辅助
└── Utils.* / AuditLogger.*       路径、UTF-8、哈希和日志工具
```

`Manifest`、`TransferPlanner` 和 `ResumeManager` 仍保留在工程中，供旧的计划/续传方案使用；当前 v5 传输主链路由 `TransferSession` 的二进制目录和文件 ID 提交状态驱动。

> 实现现状：发送端的“模式”下拉框目前没有传入传输会话；接收端的模式只有“同步镜像”会影响 v5 主链路（在提交后清理多余目标文件）。若需要严格的安全复制/跳过语义，应先将 `TransferPlanner` 的决策接入 v5 协议，而不是只修改界面文案。

运行日志写入可执行文件同级的 `logs/` 目录，编码为 UTF-8。
